# 🧠 Fern — система обучения

Проектный документ и спецификация «умнейшей» системы изучения языков по карточкам.
Цель: собрать **все режимы, что есть у конкурентов**, объединить их вокруг
**одного сильного планировщика (FSRS)** и адаптивного алгоритма «Учить», и подать
всё в крупном красивом Material 3 (ДНК ScoreMaster).

---

## 1. Что есть у конкурентов (инвентаризация)

| Приложение | Что делает | Забираем |
|---|---|---|
| **Anki** | SM-2/FSRS, типы карт (базовая, реверс, cloze, ввод, image occlusion), оценка Again/Hard/Good/Easy | FSRS-ядро, все типы карт, честная оценка |
| **Quizlet** | Flashcards, **Learn** (адаптивно MC→ввод), **Test** (генерит тест), **Match** (подбор на скорость), **Write**, **Spell** (аудио→ввод), Gravity | режимы Learn/Test/Match/Write/Spell |
| **Duolingo** | MC-перевод, tap-пары, аудирование, произношение (микрофон), **сборка фразы из слов**, стрик, XP, цель дня | сборка фразы, аудио, стрик/XP/цель |
| **Memrise** | классический повтор, **speed review**, «сложные слова», аудирование, видео-носители | speed review, «трудные слова» |
| **Busuu / Babbel** | грамматика, диалоги, уровни CEFR | (волна 3: грамм-заметки, уровни) |

**Вывод:** ни у кого нет пересечения «сильный SRS (FSRS) + богатый набор упражнений
+ нативный крупный M3 + офлайн + русский UX». Это и есть ниша Fern.

## 2. Режимы Fern (что запускается с экрана колоды)

Каждая колода открывается «пусковой панелью режимов» (крупные M3-плитки):

1. **Учить** (⭐ главный, адаптивный) — умный микс: новые карты вводятся мягко
   (узнавание → MC → ввод → вспоминание), вперемешку с due-повторами по FSRS.
   Один вход «учись» — система сама решает, что и как показать.
2. **Карточки** — классический флип-повтор (Again/Hard/Good/Easy), чистый SRS по FSRS.
3. **Тест** — генерируемый тест: выбираешь типы (MC / верно-неверно / ввод / подбор)
   и число вопросов → прохождение → **экран результатов** с разбором ошибок.
4. **Подбор** — игра на скорость: сетка слов и переводов, тапай пары, таймер + рекорд.
5. **Письмо** — только ввод перевода (строгая проверка, с опечатко-толерантностью).
6. **Аудио** — TTS произносит слово → выбрать/ввести (Spell). *(волна 2: нужен TTS)*
7. **Трудные слова** — сессия только из карт с высоким числом lapses/низкой стабильностью.
8. **Быстрый повтор** (speed) — короткая разминка: только узнавание, лёгкий таймер.

Направление в любом режиме переключается: **термин→перевод** и **перевод→термин**
(и «смешанно»).

## 3. Умное ядро — FSRS

**FSRS** (Free Spaced Repetition Scheduler) — актуальный стандарт, сильно точнее SM-2.
Реализуем сами (дефолтные веса `w[0..18]`), без внешнего пакета, чтобы контролировать
API и не зависеть от чужих релизов.

Состояние карты:
- `stability S` — сколько дней держится память до падения до `requestRetention`.
- `difficulty D` (1..10) — насколько карта «упрямая».
- `state` — new / learning / review / relearning.
- `due`, `lastReview`, `reps`, `lapses`.

Формулы (кратко):
- Retrievability `R(t) = (1 + FACTOR·t/S)^DECAY`, где `DECAY=-0.5`, `FACTOR=19/81`.
- Начальные `S = w[rating-1]`, `D = w[4] − exp(w[5]·(rating−1)) + 1` (clamp 1..10).
- При повторе: `D' = clamp(w[7]·initD(4) + (1−w[7])·(D − w[6]·(g−3)))`.
- Успех: `S' = S·(1 + exp(w[8])·(11−D)·S^(−w[9])·(exp(w[10]·(1−R))−1)·hard/easy-mod)`.
- Провал (`again`): `S' = w[11]·D^(−w[12])·((S+1)^w[13] − 1)·exp(w[14]·(1−R))`.
- Следующий интервал: `I = S/FACTOR·((requestRetention)^(1/DECAY) − 1)`, округляем,
  клампим `[1, maxInterval]`. `requestRetention` по умолчанию **0.9**.
- **Learning steps** для новых/relearning: `1 мин`, `10 мин` (внутри дня), потом «выпуск»
  в review с интервалом из FSRS.

Оценки: `again(1) / hard(2) / good(3) / easy(4)`. Свайп: влево=again, вправо=good.
Оптимизатор персональных весов — **волна 3** (нужно ≥~1000 логов; логи пишем уже сейчас
в `ReviewLog`).

## 4. Адаптивный алгоритм «Учить»

Каждой карте помимо FSRS-состояния присваивается **фаза владения**:
`unseen → recognize → produce → recall → mastered`.

Выбор упражнения по фазе (и по недавним ошибкам):
- **recognize** — MC «термин → выбери перевод» (4 варианта, дистракторы из той же колоды).
- **produce** — MC «перевод → выбери термин» или сборка фразы (для примеров).
- **recall** — ввод перевода / флип с само-оценкой.
- Ошибся → фаза откатывается на шаг, карта переигрывается позже в этой же сессии
  (короткая «learning»-очередь).
- Верно N раз подряд на фазе → повышение фазы; на `recall` верно → карта уходит в FSRS
  как выученная (`good`).

Билдер очереди сессии:
1. Берём **due**-карты (FSRS `due ≤ now`) — приоритет 1.
2. Добираем **новыми** до дневной цели (`dailyGoal`, по умолч. 20).
3. **Интерливинг**: перемешиваем так, чтобы одна карта не шла подряд и чтобы
   термин→перевод и перевод→термин чередовались.
4. «Learning»-очередь (ошибки/новые на первых шагах) — короткие возвраты внутри сессии.
5. Лимит длины сессии настраивается; прогресс — **wavy M3-progress** сверху.

## 5. Типы упражнений (спецификация виджетов)

Все — крупные M3, единый каркас `ExercisePrompt → Answer → Feedback`.

| Тип | Вопрос | Ответ | Проверка |
|---|---|---|---|
| **Flip** | термин (+пример) | тап «показать» → Again/Hard/Good/Easy | само-оценка |
| **MultipleChoice** | термин | 4 плитки-варианта | тап; верный зелёный, неверный красный + подсветка верного |
| **TypeAnswer** | термин/перевод | текстовое поле | нормализация (регистр/пробелы/диакритика), толерантность к 1 опечатке |
| **TrueFalse** | «термин = перевод?» | Верно / Неверно | булево |
| **Match** | сетка 5×2 (слова/переводы) | тап пары | все пары → время; рекорд колоды |
| **Assemble** | перевод фразы | тайлы-слова примера в нужном порядке | сравнение последовательности |
| **Listen/Spell** | 🔊 аудио (TTS) | выбрать/ввести | как MC/Type *(волна 2)* |

Общие правила UX:
- Ответ → мгновенный цвет + гаптик + (опц.) звук; авто-переход через ~700 мс или по тапу.
- «Не знаю» всегда доступно (= again).
- Один и тот же ответ карты влияет и на FSRS, и на фазу владения.

## 6. Геймификация (мягкая, из Togetherly-инфры)

- **Стрик** дней подряд с выполненной целью; **цель дня** (кол-во повторов), кольцо прогресса.
- **XP** за ответы (верный больше), недельный график.
- **Экран «Прогресс»**: стрик, кольцо цели, heatmap активности, «выучено/на изучении/новые»,
  «прогноз нагрузки» (сколько придёт на повтор в ближайшие дни), топ трудных слов.
- Локальные уведомления «N к повтору» — волна 2.

---

## 7. Дизайн (крупный M3, ДНК ScoreMaster)

- Верхний **баннер языка** (как «Игры: 101») — фильтрует колоды по изучаемому языку.
- **Сетка колод** — формы-обложки (круг/звезда/гексагон…), бейдж «N к повтору».
- **Пусковая панель режимов** на экране колоды — крупные плитки с иконкой и подписью.
- Экран упражнения: одна карта во весь экран, крупный термин (Unbounded), wavy-прогресс,
  плавные переходы, flip-анимация, гаптики.
- Нижняя навигация — зелёный круг-индикатор. FAB — быстрое добавление карты.
- Тёмная зелёная тема + Material You (dynamic color) + палитры/AMOLED.
