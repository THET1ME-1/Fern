# Бот продажи Fern Pro

Продаёт Pro за звёзды Telegram и выдаёт подписанный ключ. Нужен потому, что
Google Play не принимает платежи из России, а звёзды там покупаются и картой
«Мир», и через СБП.

Приложение проверяет ключ офлайн — сервер в схеме не участвует, бот только
подписывает и помнит, кому что выдал.

## Как работает ключ

```
FERN + base32( payload‖подпись )
       payload = версия(1) sku(1) номер(4) день выдачи(2)
       подпись = Ed25519 над payload
```

Приватный ключ живёт только у владельца бота: `~/.config/fern/license_ed25519.key`
(права 600). Публичная половина вшита в приложение —
`app/lib/services/license_service.dart`, константа `publicKeyBase64`.

Ключ **не привязан к устройству**: покупатель ставит Pro на все свои телефоны.
Утёкший ключ отзывается по номеру — номер вносится в `revoked` в том же файле и
уезжает с ближайшим обновлением.

## Запуск

```bash
python3 -m venv venv && ./venv/bin/pip install -r requirements.txt
export FERN_BOT_TOKEN=...        # от @BotFather
export FERN_BOT_OWNER=...        # ваш telegram id: журнал, возвраты
./venv/bin/python bot.py
```

Переменные:

| Переменная | Смысл | По умолчанию |
|---|---|---|
| `FERN_BOT_TOKEN` | токен бота | обязательна |
| `FERN_BOT_OWNER` | id владельца: `/stats`, `/refund`, уведомления о покупках | 0 |
| `FERN_BOT_PRICE` | цена в звёздах | 300 (≈ 4 $) |
| `FERN_BOT_DB` | журнал покупок | `bot/purchases.db` |
| `FERN_LICENSE_KEY_PATH` | приватный ключ подписи | `~/.config/fern/license_ed25519.key` |

На сервере — `fern-bot.service` (systemd): положить в `/etc/systemd/system/`,
подправить пути и `systemctl enable --now fern-bot`.

## Команды

- `/buy` — выставить счёт
- `/key` — прислать уже купленный ключ ещё раз (люди теряют переписку)
- `/stats` — сводка, только владельцу
- `/refund <номер>` — вернуть звёзды, только владельцу

## Выдать ключ руками

Пригодится для тестера, блогера или того, кто заплатил другим способом:

```bash
python3 bot/license.py --id 1001          # ключ №1001
python3 bot/license.py --verify FERN…     # проверить чужой ключ
python3 bot/license.py --pubkey           # публичный ключ (сверить с приложением)
```

Номер берите за пределами журнала бота (например, с 100000), чтобы он не
столкнулся с автоинкрементом покупок.

## Деньги

Звёзды выводятся владельцем через [Fragment](https://fragment.com): минимум
1000 звёзд, выдержка 21 день, дальше — TON. Бот к выводу отношения не имеет.

## Возврат

`/refund` возвращает звёзды покупателю, но **не отзывает ключ** — он уже у него
на руках. После возврата номер лицензии надо внести в `revoked` в
`license_service.dart` и выпустить обновление.
